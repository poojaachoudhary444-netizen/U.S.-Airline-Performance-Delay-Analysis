# ✈️ U.S. Airline Performance & Delay Analysis

## 📌 Project Overview
Flight delays and cancellations impact passengers, airline operations, and overall efficiency. This project analyzes airline performance data to identify delay patterns, cancellation trends, and operational bottlenecks using PostgreSQL and Power BI.

## 🎯 Business Objectives
- Understand the major causes of flight delays and cancellations.
- Compare airline and airport performance.
- Analyze delay trends by month, day, and time of day.
- Identify high-risk routes and airports.
- Provide recommendations to improve operational efficiency and passenger experience.

## 🛠 Tools & Technologies
- PostgreSQL
- Power BI
- Excel
- SQL
- Data Visualization

## 📂 Project Workflow

### 1. Data Ingestion
- Created tables for Flights, Airlines, and Airports.
- Imported CSV datasets into PostgreSQL.
- Verified data quality and row counts.

### 2. Data Cleaning
- Converted date fields into proper date format.
- Handled missing values.
- Standardized time fields.
- Created derived columns:
  - Flight Date
  - Delay Status
  - Time of Day
  - Cancellation Reason Description

### 3. Data Analysis
Performed SQL analysis to identify:
- Flight volumes
- Cancellation rates
- Delay patterns
- Airline performance
- Airport performance
- Route performance

### 4. Dashboard Development
Built interactive Power BI dashboards with KPIs, slicers, and visualizations.

# 🔍 Key Insights

- Evening and night flights experience the highest delays.
- Certain airports consistently show higher departure delays.
- Weather and late aircraft delays are major contributors to disruptions.
- Peak delay periods vary significantly across months.
- A small number of routes account for a significant share of delays.

---

# 💡 Recommendations

- Improve scheduling during peak delay periods.
- Enhance weather disruption planning.
- Optimize aircraft turnaround times.
- Allocate additional resources at high-delay airports.
- Monitor high-risk routes more closely.
